coin changing

Unlike the previous coin changing problem, we here have denominations that are not divisible to each other, say {1,5,6}. How we gonna deal with it?

coin_change(x){
    dp[0..x] = infinity
    dp[0] = 0;
    for i in [0..x]
        dp[i] = min(dp[i-1] + 1, dp[i-5] + 1, dp[i-6] + 1)
        // simply see negative index as infinity
}

edit distance

the minimum number of operations to transform one stirng into another. The operation includes: match, mismatch, insertion, deletion.

edit_distance(A[0..m], B[0..n]){
    for i in [0..m]
        dp[i,0] = i;
    for i in [0..n]
        dp[0,i] = i;
    for j in [1..m]
        for i in [1..n]
            match = dp[i-1, j-1];
            mismatch = dp[i-1, j-1] + 1;
            insertion = dp[i, j-1] + 1;
            deletion = dp[i-1, j] + 1;
            if(A[j] == B[i])
                dp[i,j] = min(match, insertion, deletion);
            else
                dp[i,j] = min(mismatch, insertion, deletion);
    return dp[n, m];
}

The next question is, how do we retrive the optimal alignment?

output_alignment(i, j){
    if(i == 0 and j == 0)
        return;
    if(dp[i, j] == dp[i-1, j] + 1){
        output_alignment(i-1, j);
        print "deletion";
    }
    else if(dp[i, j] == dp[i, j-1] + 1){
        output_alignment(i, j-1);
        print "insertion";
    }
    else{
        output_alignment(i-1, j-1);
        print "match \ mismatch";
    }
}

knapsack

knapsack w/ repetition: $ dp[w, i] = max_{i \leq w}(dp[w - wi] + val[i]) $.
knapsack w/o repetition: $ dp[w, i] = max(dp[w, i - 1], dp[w - wi, i - 1] + val[i]) $.

knapsack(W){
    dp[0..W, 0..n] = 0;
    for i in [0..n]{
        for w in [0..W]{
            if wi < w
                dp[w, i] = max(dp[w - wi, i - 1] + val[i], dp[w, i - 1]);
            else
                dp[w, i] = dp[w, i - 1];    
        }
    }
    return dp[W, n];
}

faster implemetation: use hashtable to store dp[w].

placing parentheses

Given a mathematical expression, output its maximum value by placing parentheses in different places. E.g., given 5 - 8 + 7 * 4 - 8 + 9, output 130. placing parentheses(d1op1d2op2...dn){
    dp[0..n, 0..n] = 0;
    for len in [0..n]{
